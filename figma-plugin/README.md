# Open Icons — Figma plugin

Search ~18k open-source icons and insert any of them into Figma as editable
vector layers. Powered by the [Open Icons](https://open-icons.vercel.app)
index — **no icons are bundled**; SVGs are fetched live from the jsDelivr CDN at
insert time, and the searchable index is loaded from the deployed website (so it
stays in sync automatically).

## Build

```bash
cd figma-plugin
npm install
npm run build      # → dist/code.js + dist/ui.html
```

## Run it in Figma

1. Figma desktop → **Plugins → Development → Import plugin from manifest…**
2. Select `figma-plugin/manifest.json`.
3. Run **Open Icons** from the Plugins menu. It opens on a live grid of the
   selected library — click any icon to drop it on the canvas.

## Features

- **Browse-first** — opens showing the selected library's icons (default
  Phosphor) in a responsive grid that reflows when you resize the window
  (drag the bottom-right corner). Scroll to load more.
- **Library + variant selectors** — pick any of the 13 libraries (or *All*) and,
  for multi-style packs, the variant (outline/solid, weights, duotone…). The
  whole grid + inserts use the chosen variant.
- **One-click Insert / Replace** — a segmented toggle that auto-switches to
  *Replace* when you have a selection and *Insert* when you don't (overridable).
  No per-icon decision.
- **Favorites & recents** — hover an icon and tap ★; switch the home grid with
  the **[▦ · ↺ · ★]** view toggle. Persisted via `figma.clientStorage`.
- **Identify a selection** — select an icon and the plugin says which library
  icon it is, most → least reliable:
  1. exact, if inserted by the plugin (`setPluginData` stamp);
  2. by the **`library/icon` layer name** (e.g. `phosphor/house`) — toggleable;
  3. **by shape**, via a geometry fingerprint (`src/lib/fingerprint.ts`, served
     from `/data/fingerprints.json`) — closest matches with a %.
- **Swap in place** — replace the selected icon with another library's version,
  preserving position and size. **Multi-select** several library icons and
  *swap them all* to a chosen library (shows how many matched).
- **Settings** — toggle layer naming and shape detection. (No data-source URL —
  the index loads from the deployed site.)

> Shape matching needs `public/data/fingerprints.json`, generated by
> `npm run sync:fingerprints` (in the website project) and committed (it isn't
> regenerated on every deploy). It's a 4–5 MB file the plugin lazy-loads only
> when you select an unrecognized vector.

## How it shares code with the website

`src/ui.tsx` imports `cdnUrl` directly from the website's `src/lib/sources.ts`,
so CDN URL construction is defined once and reused. The controller
(`src/code.ts`) runs in Figma's sandbox and only handles inserting the SVG
(`figma.createNodeFromSvg`) and persisting settings — all network calls happen
in the UI iframe.
